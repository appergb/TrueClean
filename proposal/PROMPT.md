# TrueClean · TRAE Work 创意产物提示词

> 用于在 TRAE Work（Auto 模式）中生成大赛报名所需的「创意产物 HTML」。
> 把下方【完整提示词】整段粘贴到 TRAE Work 对话框即可。

---

## 一、提示词设计总结

这份提示词专为 **TRAE AI 创造力大赛报名阶段** 设计，目标是让 TRAE Work 一次性生成一份**精美、自包含、可直接上传到报名帖**的创意产物 HTML。设计要点如下：

### 1. 角色定位
把 AI 定位为「创意提案设计师」，明确它的产出物是**单文件 HTML 创意产物**，而非代码项目或普通文字说明——避免产出偏离。

### 2. 结构化创意信息注入
把 TrueClean 的核心信息以结构化字段喂给 AI（名称/赛道/形态/定位/技术栈/痛点/三大能力/9 个 AI 工具/目标用户/价值/安全设计），AI 无需自行脑补，产出稳定可控。

### 3. 严格的设计规范
- **色彩**：指定 OKLCH 色彩空间的 teal→green 主色（accent `oklch(64% 0.15 195)`、good `oklch(66% 0.17 150)`），寓意"健康/已清理"，与桌面应用设计令牌一致。
- **主题**：要求支持深浅双主题切换 + localStorage 记忆，深色背景 `oklch(18% 0.012 255)`，浅色 `oklch(97.5% 0.005 240)`。
- **字体**：系统字体栈（Inter / PingFang SC / Microsoft YaHei），避免外部依赖。
- **质感**：premium system-utility 桌面应用质感，明确"不要模板感"。

### 4. 固定页面结构
规定 12 个区块的顺序与内容，确保报名帖审核维度（创意介绍/目标用户/价值意义）全覆盖：
1. 顶部导航条（品牌 + 赛道标签 + 主题切换）
2. Hero 区（大标题 + slogan + 关键数据 + 模拟应用窗口视觉）
3. 痛点区（不敢删/看不懂/没人帮判断）
4. 三大核心能力（看清楚/清得安全/AI 帮判断）
5. 功能详解（8 个功能 2 列网格）
6. AI Agent 亮点区（文字 + 工具标签云 + 模拟对话）
7. 目标用户（4 类用户）
8. 价值与意义（效率/社会/商业）
9. 技术架构（后端/前端/AI 三列）
10. 安全设计（2×2 分级网格）
11. 路线图（已完成/待完善）
12. Footer（参赛信息 + 技术栈）

### 5. 交互与工程约束
- 滚动渐入动画（IntersectionObserver）
- 卡片 hover 上浮、按钮 hover 变色
- 尊重 `prefers-reduced-motion`
- 响应式（≤880px 单列）
- **必须自包含**：CSS/JS 全部内联，不引用任何 CDN 或外部资源——保证离线可打开、上传后可直接预览

### 6. 输出格式约束
要求直接输出 `<!DOCTYPE html>` 到 `</html>` 的完整内容，**不包裹 markdown 代码块、不额外解释**——这样生成的结果可直接保存为 .html 文件。

### 7. 语气约束
全中文、专业克制、有质感；emoji 不滥用，每个都要有明确表意——避免 AI 常见的浮夸感。

## 二、完整提示词

```
你是 TRAE AI 创造力大赛的创意提案设计师，擅长把一个产品创意做成一份精美、自包含、可直接在浏览器打开的 HTML 创意产物页面，用于大赛报名展示。

# 任务
根据我提供的创意信息，生成一份单文件 HTML（内联 CSS，无外部依赖，可离线打开），完整展示我的创意方案。

# 创意信息
- 创意名称：TrueClean
- 参赛赛道：学习工作
- 产品形态：跨平台桌面应用（macOS/Windows/Linux）
- 一句话定位：AI 驱动的磁盘清理专家 —— 看清楚 · 清得安全 · 让 AI 帮你判断
- 技术栈：Tauri 2 + React 18 + TypeScript + Rust，内置 AI Agent（Claude/OpenAI/Ollama 多模型可选）
- 核心痛点：硬盘满了不知道被什么吃掉、也不敢乱删；现有工具要么黑盒不敢用、要么列表看不懂、要么只会扫不会想
- 三大核心能力：
  1. 看清楚：并行扫描磁盘，11 类分类占比，Treemap + Sunburst 双可视化 + 文件树下钻
  2. 清得安全：自动识别缓存/日志/临时/浏览器缓存/开发缓存/回收站，区分"绝对安全可删"与"需确认"，默认走回收站
  3. AI 帮判断：内置 AI Agent 真正调用扫描/清理工具（9 个工具：list_volumes/scan_directory/scan_junk/find_large_old_files/find_duplicates/list_applications/list_startup_items/clean_paths/empty_trash），基于真实数据给"可立即清理/建议复核/不要动"分组建议
- 目标用户：开发者、设计师、内容创作者、办公人群
- 价值：效率提升（AI 分组建议省去人工判断）+ 社会价值（安全兜底让普通用户敢自己维护电脑）
- 安全设计：默认走回收站、二次确认、系统路径红线（/System、/usr、C:\Windows 等）、API Key 本地存储不上传

# HTML 设计要求
1. 风格：premium system-utility 桌面应用质感，不要模板感。主色用 teal→green 渐变（OKLCH 色彩空间：accent oklch(64% 0.15 195)，good oklch(66% 0.17 150)），寓意"健康/已清理"。
2. 支持深浅双主题切换（右上角按钮，localStorage 记忆），深色背景 oklch(18% 0.012 255)，浅色背景 oklch(97.5% 0.005 240)。
3. 字体用系统字体栈：Inter, -apple-system, "PingFang SC", "Microsoft YaHei", system-ui。
4. 页面结构（按顺序）：
   - 顶部导航条（品牌名 + 赛道标签 + 主题切换）
   - Hero 区：大标题 + slogan + 核心定位 + 关键数据 + 一个模拟应用窗口的视觉（用纯 CSS 画一个带标题栏的窗口，里面放磁盘使用率卡片 + 简易 treemap 色块 + AI 对话气泡）
   - 痛点区：3 个痛点卡片（不敢删/看不懂/没人帮判断）
   - 三大核心能力：3 列卡片，每列带编号、标题、描述、要点列表
   - 功能详解：2 列网格，8 个功能（概览/扫描可视化/系统垃圾/大文件/重复文件/应用卸载/启动项/设置）
   - AI Agent 亮点区：左侧文字 + 工具标签云，右侧模拟一段 AI 对话（用户问"能清理多少"→ AI 调用 scan_junk → 输出三组分组建议）
   - 目标用户：4 列卡片（开发者/设计师/创作者/办公人群）
   - 价值与意义：3 列卡片（效率提升/社会价值/商业潜力）
   - 技术架构：3 列（桌面框架后端 / 前端可视化 / AI Agent）
   - 安全设计：2×2 网格（绝对安全可删/需确认/绝不要动/兜底机制），用左边框颜色区分（绿/黄/红/蓝）
   - 路线图：2 列（已完成 / 待完善），已完成用 ✓，待完善用 ○
   - Footer：参赛信息（赛事/赛道/创意名称/产品形态）+ 技术栈 + 报名阶段日期
5. 交互：滚动渐入动画（IntersectionObserver）、卡片 hover 上浮、按钮 hover 变色、尊重 prefers-reduced-motion。
6. 响应式：≤880px 时多列布局变单列。
7. 全中文，专业、克制、有质感的语气。不要用 emoji 滥情，每个 emoji 都要有明确表意。
8. 必须自包含：所有 CSS 内联在 <style>，所有 JS 内联在 <script>，不引用任何 CDN 或外部资源。

# 输出
直接输出完整 HTML 文件内容，从 <!DOCTYPE html> 开始到 </html> 结束，不要包裹在 markdown 代码块里，不要额外解释。
```

## 三、使用方式

1. 打开 TRAE Work，选择 **Auto 模式**。
2. 把上方【完整提示词】整段粘贴到对话框，发送。
3. 等待 3–5 分钟，TRAE Work 会生成完整 HTML。
4. 把生成结果保存为 `trueclean-proposal.html`，本地双击预览。
5. 确认无误后上传到报名帖。

> 提示：若生成结果想微调（如换配色、加区块），可在 TRAE Work 里继续对话追加要求，例如"把主色换成蓝紫渐变"或"加一个 FAQ 区块"。
