## 1. 安装

### 系统要求

| 平台 | 要求 |
|---|---|
| macOS | 11.0 (Big Sur) 及以上 |
| Windows | 10 / 11（64 位） |
| Linux | 现代发行版（需 webkit2gtk，详见 Tauri 前置依赖） |

### 从发布版安装

1. 前往 GitHub Releases 页面（1.0 发布后可用）。
2. 下载对应平台的安装包：
   - macOS：`.dmg`（拖入 Applications）
   - Windows：`.msi` 或 `.exe`
   - Linux：`.AppImage` 或 `.deb`
3. 安装后启动 TrueClean。

> **macOS 首次打开**：若提示"无法验证开发者"，右键 → 打开，或到「系统设置 → 隐私与安全性」点击「仍要打开」。

### 从源码构建（开发者）

前置：Rust（stable）、Node ≥ 18、[pnpm](https://pnpm.io)。

```bash
git clone https://github.com/appergb/TrueClean.git
cd TrueClean
pnpm install
pnpm tauri dev      # 开发模式
pnpm tauri build    # 产出安装包
```

详见 [CONTRIBUTING.md](../CONTRIBUTING.md)。

## 2. 首次启动

启动后你会看到**概览页（Overview）**：

- 顶部显示每个磁盘卷的**使用率环形图**、总容量 / 已用 / 可用空间。
- 醒目的「开始扫描」入口。
- 各功能引导卡片。

左侧**侧边栏**是导航：概览 / 扫描 / 系统垃圾 / 大文件 / 重复文件 / 应用卸载 / 启动项 / 设置。
右侧可切出**AI 助手抽屉**（点顶部 AI 按钮或侧边栏图标）。

## 3. 磁盘扫描与可视化

### 扫描步骤

1. 点击侧边栏「扫描」或概览页「开始扫描」。
2. 选择要扫描的**磁盘卷**或**目录**（用系统对话框选）。
3. 扫描开始，实时显示：已扫描文件数 / 字节数 / 当前路径，可随时点「取消」。
4. 扫描完成后展示：
   - **分类占比条**：11 类（系统/应用/开发/文档/媒体/缓存/日志/废纸篓/下载/压缩包/其他）的体积与百分比。
   - **矩形树图（Treemap）**：每个矩形代表一个目录/文件，面积=体积，颜色=类别。点击可下钻，悬停看详情。
   - **旭日图（Sunburst）**：同心环展示层级，中心是根，外层是子目录。点击下钻。
   - **文件树**：从根逐层进入最大子目录，带面包屑返回。

### 交互技巧

- **下钻**：在 Treemap/Sunburst 上点击某个区块，进入该目录的细分视图。
- **返回**：用文件树顶部的面包屑回到上层。
- **找大块**：Treemap 上最大的矩形就是最占空间的目录，顺藤摸瓜。

> 扫描大目录（如整个磁盘）可能需要几十秒到几分钟，取决于文件数量。扫描结果会缓存，下次同目录未变化时可复用。

## 4. 系统垃圾清理

### 功能说明

自动识别 9 类可清理垃圾：

| 分组 | 示例 | 安全等级 |
|---|---|---|
| 用户缓存 | `~/Library/Caches` | 安全可删 |
| 系统缓存 | `/Library/Caches` | 安全可删 |
| 应用缓存 | 各 App 缓存 | 安全可删 |
| 日志文件 | `~/Library/Logs` | 安全可删 |
| 临时文件 | `/tmp`、系统 Temp | 安全可删 |
| 浏览器缓存 | Chrome/Safari/Edge/Brave/Firefox | 安全可删 |
| 开发缓存 | Xcode DerivedData、node_modules 缓存 | 安全可删 |
| 语言缓存 | npm/cargo/pip/gradle/maven | 安全可删 |
| 回收站 | 已删除内容 | 清空不可恢复 |

### 清理步骤

1. 点击侧边栏「系统垃圾」。
2. 点击「扫描垃圾」，等待分组列出。
3. 每组显示：体积、项数、是否「推荐清理」。
4. **勾选**要清理的组（或组内单项）。可组级全选。
5. 底部实时汇总「已选 N 项 / 预计释放 X」。
6. 点「清理」→ 弹出**确认框**：显示将删除项数、释放空间、是否进回收站。
7. 确认后执行。默认**进回收站**（可恢复）。结果以 Toast 显示。

> 开发缓存（如 Xcode DerivedData、npm cache）通常占几十 GB，清理后系统/工具会自动重建，是释放空间的主力。

## 5. 大文件查找

### 功能说明

按「最小大小」+「至少多少天未修改」筛出体积大且陈旧的文件，帮你发现可能可以归档或删除的大文件。

### 使用步骤

1. 点击侧边栏「大文件」。
2. 设置筛选条件：
   - **最小大小**：如 100MB（只看比这大的）。
   - **至少多少天未修改**：如 180 天（只看半年没动过的）。
3. 选择搜索目录，点「查找」。
4. 结果按大小排序，显示路径、体积、最后修改时间。
5. 勾选要删除的 → 底部汇总 → 「删除」→ 确认（默认进回收站）。

> 大文件属于「需用户确认」类别——可能是你珍视的视频/备份，删除前务必逐项确认。

## 6. 重复文件去重

### 功能说明

基于 **blake3 内容哈希**精确去重（不是按文件名，是按文件内容）。先按大小分桶，再对同大小文件哈希，速度快且不误判。

### 使用步骤

1. 点击侧边栏「重复文件」。
2. 选择搜索目录，设置参与去重的最小文件大小（如 1MB，忽略小文件）。
3. 点「扫描」，等待完成。
4. 结果按组展示：每组是内容相同的多个文件，显示每组的浪费空间（= (数量-1) × 单文件大小）。
5. 每组**默认保留 1 个**，其余勾选可删。你可调整保留哪个。
6. 勾选 → 汇总 → 「删除」→ 确认（默认进回收站）。

> 重复文件去重是释放空间的利器——摄影师/设计师的多次备份常产生大量重复。

## 7. 应用卸载

### 功能说明

列出已安装应用，卸载时**连带清理残留**（缓存、偏好设置、Application Support、日志），而不是只删主程序。

### 使用步骤

1. 点击侧边栏「应用卸载」。
2. 列出所有应用：名称、版本、体积、最近使用时间。
3. 找到要卸载的 App，点「卸载」。
4. 确认框显示将删除的主程序 + 残留路径 + 释放空间。
5. 确认后执行（默认进回收站）。卸载报告显示已删路径与残留供复核。

> 卸载属于「需用户确认」操作。残留清理会查找 `~/Library/Application Support/<App>`、`~/Library/Caches/<App>`、`~/Library/Preferences/<App>.plist` 等，受保护路径约束（不会动系统组件）。

## 8. 启动项管理

### 功能说明

列出开机自启动项（登录项 / 启动代理 / 服务），可一键启停，加快开机速度。

### 使用步骤

1. 点击侧边栏「启动项」。
2. 列出所有启动项：名称、路径、类型（launchAgent/loginItem/registry/autostart）、启用状态。
3. 用开关启停某项。变更即时生效。

> 启停启动项是相对安全的操作（不删除文件，只改启用状态）。但禁用某些系统服务可能影响功能，不确定的项建议先搜索用途。

## 9. 配置 AI 助手

TrueClean 内置 AI 助手，能基于你的**真实磁盘数据**给清理建议。需先配置 Provider。

### 配置步骤

1. 点击侧边栏「设置」。
2. 选择 **Provider**：
   - `claude`（默认，质量高，需 API Key）
   - `openai`（通用，需 API Key）
   - `ollama`（本地运行，零成本，隐私优先）
3. 填写对应凭据：
   - Claude / OpenAI：填 **API Key**（密码态输入，可切换显示）。
   - Ollama：填 **地址**（默认 `http://localhost:11434`）。
4. 填 **Model**（如 `claude-sonnet-4-6`、`gpt-4o`、`llama3.1`）。
5. 可选：点「测试连接」验证配置是否有效。
6. 点「保存」。

### 获取 API Key

- **Claude**：console.anthropic.com 注册 → API Keys。
- **OpenAI**：platform.openai.com → API Keys。
- **Ollama**：ollama.com 下载安装，本地运行 `ollama pull llama3.1`，无需 Key。

> **Key 仅保存在你本地的配置文件中**，应用不上传、不内置任何密钥。详见 [SECURITY.md](SECURITY.md)。

## 10. 使用 AI 助手

### 打开

点顶部 AI 按钮或侧边栏图标，右侧滑出 AI 助手抽屉。

### 提问示例

- 「帮我看看能安全清理多少空间」
- 「我的磁盘为什么满了？」
- 「~/Developer 目录下有哪些大文件可以清理」
- 「找出重复文件并告诉我能回收多少」
- 「哪些应用很久没用了，可以卸载」

### AI 的工作方式

1. **先调工具取真实数据**：Agent 会调用扫描/查找工具读取你的磁盘，绝不编造路径和体积。
2. **结构化建议**：按三组输出：
   - 【可立即清理】绝对安全（缓存/日志/临时），给合计释放空间。
   - 【建议复核】需你确认（大文件/应用），说明为什么需确认。
   - 【不要动】系统关键路径，主动排除并说明原因。
3. **执行需确认**：当你同意清理某项，Agent 会执行（默认进回收站）。破坏性操作前前端会弹确认。

### 工具调用可视化

对话中会显示**工具调用卡片**：展示 Agent 调了哪个工具、参数、结果摘要，可折叠。让你看清 AI 的"思考过程"，不是黑盒。

## 11. 撤销清理

### 什么时候能撤销

- 清理时**默认进回收站**（`to_trash=true`）的操作会记录快照（`CleanManifest`），可撤销。
- **永久删除**（`to_trash=false`）不可撤销——这是有意设计。

### 如何撤销

清理后如果后悔，可从回收站还原。TrueClean 记录了最近一次回收站清理的快照，支持一键还原到原路径。

> **平台差异**：
> - Windows/Linux：用系统回收站原生 API 还原，按原路径 + 时间匹配。
> - macOS：在 `~/.Trash` 按文件名定位并移回原位（同名冲突时不消歧，罕见情况）。

## 12. 安全提示

### TrueClean 的安全承诺

1. **删除默认进回收站**——可恢复。永久删除需你显式选择。
2. **系统关键路径受保护**——`is_protected` 硬编码红线，绝不删 `/System`、`/usr`、`C:\Windows` 等。
3. **所有破坏性操作二次确认**——UI 弹框显示删什么、释放多少。
4. **AI 有安全红线**——系统提示词禁止建议删系统路径；工具内部 `is_protected` 兜底。
5. **不上传文件内容**——与 LLM 只交换路径摘要 + 体积，不传文件内容。
6. **API Key 仅存本地**——不内置、不上传。

### 你应该注意的

- **永久删除不可撤销**：除非你明确需要，否则始终用「进回收站」。
- **大文件/重复文件需复核**：AI 标记为「建议复核」的可能是你有用的数据。
- **定期清空回收站**：进回收站的文件仍占空间，确认无误后可清空。
- **不要在共享电脑上保存 API Key**：配置文件是明文存储。
- **不确定就不删**：宁可保守，不要冒险。

## 13. 常见问题（FAQ）

### Q: 扫描很慢怎么办？
A: 扫描整块磁盘（尤其含大量小文件）需要时间。建议先扫特定目录（如 `~/Library` 或 `~/Developer`）。扫描结果会缓存，同目录未变化时下次秒开。

### Q: 清理后空间没释放？
A: 进回收站的文件仍占空间。确认无误后，到「系统垃圾」清空回收站，或手动清空系统废纸篓。

### Q: macOS 上撤销还原失败？
A: macOS 的还原按 `~/.Trash` 文件名匹配。如果回收站有同名文件，可能匹配错误。建议手动从废纸篓拖回。Windows/Linux 用原生 API，更可靠。

### Q: AI 助手报错"Key 无效"？
A: 检查设置里的 API Key 是否正确、是否过期、是否有余额。Claude/OpenAI 需有效 Key；Ollama 需先本地运行 `ollama serve`。

### Q: AI 助手能直接删文件吗？
A: 能，但有保护：默认进回收站；`is_protected` 拦截系统路径；前端弹确认。你拒绝则不执行。

### Q: 卸载应用后残留没清干净？
A: 残留清理基于已知路径模式（Application Support/Caches/Preferences）。某些 App 用非标准路径，残留会列出供你手动复核。

### Q: Windows/Linux 支持完善吗？
A: 1.0 基线已支持三平台，但 Windows/Linux 的清理路径表、卸载残留、启动项管理仍在打磨（见 [ROADMAP.md](ROADMAP.md)）。macOS 体验最完整。

### Q: 我的数据会被上传吗？
A: 不会。TrueClean 是纯本地应用，与 LLM 只交换路径摘要 + 体积（不含文件内容）。用 Ollama 本地模型则数据完全不出本机。详见 [SECURITY.md](SECURITY.md)。

### Q: 如何贡献？
A: 欢迎提 Issue 和 PR。详见 [CONTRIBUTING.md](../CONTRIBUTING.md)。删除相关逻辑改动请格外谨慎并补测试。
